# Contributing to CNDS - Complex Network Dynamics Simulator

Thank you for your interest in contributing to the CNDS project! Your contributions are greatly appreciated. Please follow the guidelines below to ensure a smooth process.

## Contribution Guidelines

### Code Standards
1. **Consistent Style**: Follow the coding style used in the existing codebase. Use proper indentation and naming conventions.
2. **Commenting**: Write clear, concise comments for your code, explaining the logic and purpose.
3. **Testing**: Ensure that your code is well-tested with unit tests and integration tests where applicable.

### Commit Message Format
1. **Short Summary**: Start with a brief summary of the changes (max 50 characters).
2. **Detailed Description**: Include a more detailed description of the changes, if necessary. Wrap it at 72 characters.
3. **Use Imperative Mood**: Write your commit message in imperative mood (e.g., "Fix bug in..." instead of "Fixed bug in...").

### Pull Request Process
1. **Fork the Repository**: Create a fork of the repository on GitHub.
2. **Create a Branch**: Create a new branch for your feature or bug fix.
3. **Submit a Pull Request**: Push your changes to your fork and submit a pull request to the `main` branch of the original repository.
4. **Review**: Once submitted, your pull request will be reviewed by the maintainers. You may be asked to make changes based on feedback.

Thank you for contributing! Your efforts help make this project better!
